# Задача: Сравнение скорости суммирования вектора в одном и нескольких потоках

## Описание

Программа демонстрирует разницу в производительности между **однопоточным** и **многопоточным** выполнением при подсчёте суммы элементов большого вектора.

---

## Постановка задачи

1. **Создать вектор** из `1 000 000` элементов.  
2. **Заполнить** вектор случайными числами от `1` до `100`.  
3. **Посчитать сумму** всех элементов:
   - сначала **в одном потоке** (обычный цикл);
   - затем **в нескольких потоках** (например, 4 потока, каждый считает часть массива).  
4. **Замерить время** выполнения каждого варианта.  
5. **Сравнить результаты** — показать, насколько многопоточность ускоряет вычисление.

---

## Используемые технологии

- **C++17** или новее  
- `<thread>` — для работы с потоками  
- `<vector>` — для хранения данных  
- `<chrono>` — для измерения времени выполнения  

---

## Как работает программа

1. Создаётся вектор `std::vector<int>` длиной `1 000 000`.  
2. Заполняется псевдослучайными числами по простой формуле (линейный конгруэнтный генератор).  
3. Измеряется время выполнения при подсчёте суммы **в одном потоке**.  
4. Массив делится на 4 равные части, каждая часть обрабатывается **в отдельном потоке**.  
5. Потоки возвращают частичные суммы, которые складываются в общий результат.  
6. Сравнивается время выполнения и выводится **ускорение (Speedup)**.
